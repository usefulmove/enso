## Approach & Verification Plan

### Steps
1. **Lazy load** — Locate resume source file (PDF, Word, Markdown, etc.)
2. **Identify placement** — Best section for the phrase:
   - Summary/objective: high visibility, sets tone immediately
   - Projects section: ensoOS as a project entry with description
   - Experience: if framed as founder/builder work
3. **Draft content** — Concise, founder voice but resume-appropriate:
   - Bullet point or short paragraph
   - Include "the emacs of agent orchestrators" as the descriptor
   - Include "software building software" and "ensoOS" for searchability
   - Quantify or qualify impact where possible
4. **Balance** — Visionary language must coexist with concrete accomplishments
5. **Review** — Present draft for approval
6. **Execute** — Apply to resume source file
7. **Verify** — Check formatting, layout, and tone in final rendered output
