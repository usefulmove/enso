# Session: The North Star — enso as a Fold
**Date:** 2026-04-25

## Overview

Deep architectural session exploring the scheduler primitive and formalizing enso's execution model. Started with a question about SQLite for story indexing, rejected it, and instead discovered the fold model and the missing scheduler primitive.

## Key Decisions

- Rejected SQLite for story indexing — breaks the plain-text principle, wrong abstraction for this layer
- Formalized the north star: **enso is a fold over stories, with context as the accumulator**
- Stories are `context → context` transformations; Context Scope is the function signature; Acceptance Criteria are postconditions
- The scheduler is a missing OS primitive — enso needs a process table (`QUEUE.md`) to track story state transitions
- The three-layer model (kernel/OS/interpreter) holds up when pushed to its logical conclusion
