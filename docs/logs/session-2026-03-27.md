# Session: ensoOS — Founding Session
**Date:** 2026-03-27

## Overview

First founding session. Established the vision, metaphors, and co-creator relationship
for enso as an agent operating system. Bootstrapped the harness structure and created
the first four active stories.

## Key Decisions

- **Vision named:** "the emacs of agent orchestrators" — enso as a self-extending,
  agent-programmable substrate from which agent teams are hired, sculpted, and orchestrated
- **ensoOS framing adopted:** The harness is not just scaffolding — it is an operating
  system for agents. Processes = Agents, Memory = Context, Filesystem = docs/, Programs = Skills
- **Human role defined:** "the God agent" — the technical program manager and systems
  architect who can design, hire, and sculpt any team of agents in any shape, size, focus,
  or expertise
- **Co-creator relationship established:** Human + agent as founding team. "Building with
  my brothers and sisters."
- **Multi-agent team model confirmed:** Named agents with roles (e.g., Toni the architect)
  as persistent collaborators, not transient tools
- **Key phrases coined:**
  - "participates in the construction of its own operational context"
  - "the emacs of agent orchestrators"
  - "ensoOS — the agent operating system"
  - "software building software"
  - "agents that live in a harness (mouldable environment) to solve problems"
