# Session: OKR-sharpened story contract (STORY-013)
**Date:** 2026-05-28

## Overview
Evaluated what OKR principles could add to the `enso.story/v1` contract, then landed the result. The original ask — "create a v2 STORY.md strengthened by OKRs" — collapsed under pressure-testing into a no-schema discipline layer. Key reframe: OKR's value for an agent harness is the *outcome-not-activity* discipline and binary thresholds, not grading, stretch, or cadence (those belong to human performance management and would weaken enso's auditable pass/fail loop). The user chose self-contained "egg" stories over a PRD alignment spine, which kept OKR inside the story and reinforced story sovereignty.

## Key Decisions
- **No schema bump.** `enso.story/v1` is unchanged; this is a contract-discipline clarification, not a structural change.
- **AGENTS.md protocol version 0.9.0 → 0.10.0** to signal added authoring discipline (minor bump, not a patch).
- **The contract ladder:** Objective (`## Goal`) → Key Results (`## Acceptance Criteria`) → Proof (`## Verification Contract`). No new fields — the middle rung just got a clear job.
- **Key results are binary, threshold-shaped.** Magnitude rides in a threshold (`p95 ≤ 200ms`); baselines live in the Goal as motivation; no 0.0–1.0 grading.
- **Outcome over activity** as a default, with an explicit escape hatch for trivial/mechanical stories (§2 instruction economy).
- **No stretch.** A story is sized so full success = acceptance; optional work is a separate egg.
- **No PRD alignment spine.** Eggs stay sovereign; coherence, if wanted, is reconstructed retrospectively by a curator — never via forward pointers.
