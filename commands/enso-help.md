## ENSO - Context Management Framework v0.5.0

A minimal system for managing AI agent context across sessions. Treats context as a scarce resource — load only what's needed, persist before it's lost, compact before overflow.

---

## Commands

| Command | Description |
|---------|-------------|
| `/enso-start` | Start session. Loads PRD, ARCHITECTURE, LESSONS, recent logs, and detects the active story. Bootstraps new projects on first use. |
| `/enso-close` | Close session. Drafts a session log, lessons, and story updates for your review. Writes nothing until confirmed. |
| `/enso-compact` | Compact working context. Drafts what to persist, drop, and keep active. Run when context feels heavy (~80% tokens used). |
| `/enso-log [n]` | Show last N session logs (default: 5). Read-only. |
| `/enso-help` | Show this reference. |

---

## The Six Operations

| Operation | What It Does |
|-----------|--------------|
| **Write** | Persist information outside the context window |
| **Select** | Load only what's needed right now |
| **Probe** | Actively search (grep, LSP, glob) for answers |
| **Compress** | Summarize to fit the token budget |
| **Isolate** | Split work across multiple scopes |
| **Assign** | Choose the ideal agent for each task |

## Directory Structure

```

│   ├── PRD.md             # Problem, goals, scope

├── reference/             # Long-term memory

│   └── completed/         # Finished stories
├── skills/                # Reusable procedures and scripts
└── logs/                  # Session history (auto-generated by /enso-close)
```

## Typical Session Workflow

```
1. /enso-start       → load context, confirm active story (bootstraps if new project)
2. ... do work ...
3. /enso-compact     → if context gets heavy mid-session
4. /enso-close       → capture lessons, write session log
```

---

## Best Practices

1. **Start and close every session** with `/enso-start` and `/enso-close`.
2. **Compact proactively** — don't wait for overflow; compact at ~80% context.
3. **Keep stories tightly scoped** — one goal, explicit write/read/exclude boundaries.
4. **Plan before executing** — complete the story's Approach section before touching files.
5. **Capture lessons immediately** — `/enso-close` is the moment to record insights.
6. **Build tools, don't just use them** — when you hit friction, author a skill. Software builds software.
7. **Update, don't accumulate** — edit docs in place; git preserves history.
